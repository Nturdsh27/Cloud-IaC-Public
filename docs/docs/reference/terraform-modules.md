# Terraform Module Reference

Standard interface and conventions for all Terraform modules in the kit.

## Module Sources

Reference modules from GitHub:

```hcl
module "example" {
  source = "github.com/Nturdsh27/Cloud-IaC//modules/<domain>/<module>?ref=v1.0.0"
}
```

!!! warning "Always pin to a release tag"
    Use `?ref=vX.Y.Z` in production. The `main` branch may contain breaking changes.

## Module Structure Convention

Every module follows this layout:

```
modules/<domain>/<name>/
├── main.tf           # Primary resources
├── variables.tf      # All inputs with descriptions + validation
├── outputs.tf        # All outputs with descriptions
├── versions.tf       # Required providers and Terraform version
├── README.md         # Auto-generated by terraform-docs
├── examples/
│   ├── basic/        # Minimal working example
│   └── complete/     # Full-featured example
└── tests/
    └── main.tftest   # Terraform native tests
```

## Naming Conventions

| Element | Convention | Example |
|---------|-----------|---------|
| Module directory | `kebab-case` | `landing-zone` |
| Resource names | `snake_case` | `google_compute_network.main` |
| Variable names | `snake_case` | `enable_cloud_nat` |
| Output names | `snake_case` | `network_id` |
